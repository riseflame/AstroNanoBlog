本篇blog主要介绍一下在本地部署LLM时硬件选型的一些思路和经验，在咨询了网上多个供应商后，都无法提供具体的硬件转化到部署后的Benchmark数据，本文主要是基于我自己的计算和一些网上的资料整理而成。

# 一.硬件选型

首先是根据预算确定是用什么显卡，以及卡的数量，我们的预算在80万左右，并且我们没有机房，无法使用机架式服务器，只能使用塔式服务器，塔式服务器的显卡数量一般在4-6卡，6卡需要对主机进行改造，成本较高，比较通用的还是四卡的方案。

从显卡的角度来看，符合我们预算的显卡性价比较高的就是：RTX PRO 6000 和 H100， pro6k价格在十万左右，四卡整机配下来供应商的报价在五十五万左右，H100单卡20W左右，四卡整机配下来供应商的报价九十万。

| 对比 | RTX PRO 6000 Blackwell | H100 |
|---|---|---|
| 代际 | 更新，Blackwell，2025 | 较早，Hopper，2022 |
| 显存容量 | 96GB| 80GB / 94GB |
| 显存类型 | GDDR7 ECC | HBM3 |
| 显存带宽 | 约 1.8 TB/s | 约 3.35-3.9 TB/s |
| 交火 | Pcie5.0 | NVlink |

p6k的优势显存比较大，而且架构新，支持NVFP4，和其他四位的模型精度，可以部署下更大的模型，H100的优势是显存带宽大，支持NVlink，算力也高训练和推理的prefill都有优势。

## 2.2 推理性能估算
吞吐量计算比较复杂，计算时需要考虑到多卡交火的速度，tp的数量，以及推理框架、prefill阶段速度，显存带宽等等参数，很难计算一个精准的吞吐量，下面是我根据网上资料做的估算。
下面这个是在PCIE3.0下测试的吞吐量，作者也提到在PCIE5.0下dsv4 flash能到350token/s
![image.png](https://img.zzliu.com/file/1783506276995_image.png)

根据另一份资料，吞吐量随着并发下降的趋势如下：
![image.png](https://img.zzliu.com/file/1783506495939_image.png)
简单估算一下在10并发下的tok/s是单并发下的0.7，那对于dsv4 flash在PCIE5.0下的10并发吞吐量大概是350*0.7=245tok/s。

对于glm5.2来说，如果按照1.5倍去算5.0下的推理速度，fp4大概可以到60tok/s，10并发下大概是60*0.7=42tok/s。
